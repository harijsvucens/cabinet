# Upstream Update Survival Strategy

## Current Setup: Fork + Rebase (Active)

```
origin   → https://github.com/harijsvucens/cabinet.git  (your fork, custom commits here)
upstream → https://github.com/cabinetai/cabinet.git     (upstream source, pulls from here)
```

## What Gets Destroyed

Any modification to **tracked source files** survives IF committed to your fork. Uncommitted changes get clobbered by rebase:

| Path | At-Risk? | Mitigation |
|------|----------|------------|
| `src/` | Yes — commit to fork | Contribute upstream via PR when possible |
| `server/` | Yes — commit to fork | Same |
| `electron/` | Yes — commit to fork | Same |
| `scripts/` | Yes — commit to fork | Or keep diffs as local patches |
| `cabinetai/`, `cli/` | Yes — commit to fork | Same |
| `docs/` | Yes — commit to fork | New docs are safe. Core doc edits must be re-applied. |
| `package.json` | Prefer upstream | Take upstream, re-apply custom deps manually |
| `package-lock.json` | Always upstream | Regenerated by `npm install` |
| `next.config.ts` | Yes — commit to fork | Only if absolutely necessary |
| `tsconfig.json` | Yes — commit to fork | Only if absolutely necessary |
| `CLAUDE.md` | Yes — commit to fork | Accept upstream, re-append custom instructions |
| `PROGRESS.md` | Append-only | Resolve conflicts by keeping both sides |

## Update Workflow (Fork + Rebase)

```
# AI agent: follow this exact sequence. DO NOT skip steps or improvise.

1. Stash or commit all current work
2. git fetch upstream
3. git rebase upstream/main                  # replay local commits on latest upstream
4. Resolve any conflicts
5. npm install                               # update deps if upstream changed them
6. npm run dev:all                           # verify it works
7. git push origin main                      # push rebased state to your fork
```

**Conflict resolution rules:**
- `data/` conflicts: NEVER take upstream. Always keep YOUR data.
- `skills-lock.json`: NEVER take upstream. Your skill registrations.
- `.agents/skills/`: NEVER take upstream. Your installed skills.
- `package.json`/`package-lock.json` conflicts: Take upstream, then `npm install`.
- `CLAUDE.md`: Accept upstream, then re-append custom instructions.
- `PROGRESS.md`: Accept both sides (append).
- Source file conflicts: Manually resolve. Prefer upstream for core logic, keep your changes only where needed.

**If rebase goes wrong:**

```
git rebase --abort                          # cancel the rebase
git log --oneline -10                       # review what you have locally
# Debug the conflict, try again
```

**Important: never force-push upstream.** Only force-push to `origin` (your fork). Upstream is read-only.

## General Rules for AI Agents Editing This Project

### DO
- Put all content and agent config in `data/` (gitignored, safe forever)
- Add new files under `docs/` for custom documentation (new files don't conflict)
- Use `.env.local` for all configuration (gitignored)
- Append to `PROGRESS.md` after every change (required by project convention)
- Append custom rules to end of `CLAUDE.md` (under a `## Custom Instructions` heading)
- Back up `data/` before any risky operation
- Commit source changes and push to `origin` (your fork) so they survive rebase

### DO NOT
- Edit upstream-only source without committing to your fork
- Force-push to `upstream` (cabinetai/cabinet) — read-only
- Modify `package.json`, `package-lock.json`, `tsconfig.json`, `next.config.ts` unless absolutely necessary
- Run `git push` without explicit user instruction
- Delete or overwrite anything in `data/`
- Run `npm install` that changes `package-lock.json` unless after an upstream rebase

### Emergency Recovery

```
# If data/ gets corrupted:
git -C data/ log --oneline -10    # find last good commit in data repo
git -C data/ checkout <hash> -- . # restore data/ to that commit

# If rebase is hopelessly broken:
git rebase --abort
git log --oneline origin/main..HEAD  # list your unmerged commits
# Cherry-pick them onto a clean upstream base if needed

# If source is broken after rebase:
npm install                       # restore dependencies
npm run dev:all                   # verify it works
```
